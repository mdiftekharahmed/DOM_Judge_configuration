# Domjudge Docker Setup

This guide provides instructions to set up and run a Domjudge competitive programming platform instance using Docker Compose.

## Prerequisites

Before you begin, ensure your system meets the following requirements:

1.  **Docker and Docker Compose**: Install Docker and Docker Compose if they are not already installed.
2.  **Enable cgroup memory and swap accounting**: This is crucial for the `judgehost` containers to function correctly.
    *   Open the GRUB configuration file for editing (e.g., with `sudo nano /etc/default/grub`).
    *   Locate the line starting with `GRUB_CMDLINE_LINUX_DEFAULT`.
    *   Append the necessary options:
        *   For most systems: `cgroup_enable=memory swapaccount=1`
        *   For modern Linux distributions (e.g., Debian Bullseye, Ubuntu Jammy Jellyfish) that use cgroup v2 by default: `cgroup_enable=memory swapaccount=1 systemd.unified_cgroup_hierarchy=0`
        *   Example: `GRUB_CMDLINE_LINUX_DEFAULT="quiet splash cgroup_enable=memory swapaccount=1 systemd.unified_cgroup_hierarchy=0"`
    *   Save the file and exit the editor.
    *   Update GRUB:
        ```bash
        sudo update-grub
        ```
    *   Reboot your system:
        ```bash
        sudo reboot
        ```

## Setup and Running Domjudge

Follow these steps to get your Domjudge instance up and running:

1.  **Prepare Docker Compose File**:
    *   Ensure you have the `docker-compose.yml` file in your project root directory. The provided `docker-compose.yml` is configured to run:
        *   `mariadb`: The database server.
        *   `domserver`: The main Domjudge web server.
        *   `judgehost-0`, `judgehost-1`, `judgehost-2`, `judgehost-3`: Four judge daemons. You can adjust the number of judgehosts by adding or removing similar service definitions in the `docker-compose.yml` file.

2.  **Start MariaDB and Domserver**:
    *   Open a terminal in the directory containing your `docker-compose.yml` file.
    *   Run the following command to start the database and the main Domjudge server in detached mode:
        ```bash
        sudo docker compose up -d mariadb domserver
        ```

3.  **Configure Judgehost Password**:
    *   The judgehosts need a password to connect to the domserver. This password is automatically generated by the `domserver` container.
    *   Retrieve the `JUDGEDAEMON_PASSWORD` by running:
        ```bash
        sudo docker exec -it domserver cat /opt/domjudge/domserver/etc/restapi.secret
        ```
    *   Copy the outputted password.
    *   Open your `docker-compose.yml` file and update the `JUDGEDAEMON_PASSWORD` environment variable for **each** `judgehost` service with the password you just copied.
        For example, if the password is `MySecretPassword123`, change:
        ```yaml
        # ...
        services:
          # ...
          judgehost-0:
            # ...
            environment:
              - DAEMON_ID=0
              - JUDGEDAEMON_PASSWORD=J5ixWQ7y+6P80udmnfHhfE0XFilD9vn/ # <-- Replace this
          # ...
        ```
        to:
        ```yaml
        # ...
        services:
          # ...
          judgehost-0:
            # ...
            environment:
              - DAEMON_ID=0
              - JUDGEDAEMON_PASSWORD=MySecretPassword123 # <-- With the new password
          # ...
        ```
        **Repeat this for all `judgehost` entries (`judgehost-1`, `judgehost-2`, `judgehost-3`).**

4.  **Start Judgehosts**:
    *   Once the `JUDGEDAEMON_PASSWORD` is updated in your `docker-compose.yml` file for all judgehosts, you can start all services, including the judgehosts:
        ```bash
        sudo docker compose up -d
        ```
        This command will also start `mariadb` and `domserver` if they are not already running, and it will create and start the `judgehost` containers.

5.  **Access Domjudge**:
    *   Your Domjudge instance should now be accessible via a web browser.
    *   Open: [http://localhost:12345](http://localhost:12345) (or replace `localhost` with your server's IP address if accessing remotely).
    *   To get the initial administrator password, run:
        ```bash
        sudo docker exec -it domserver cat /opt/domjudge/domserver/etc/initial_admin_password.secret
        ```
    *   Use this password to log in as `admin`.

## Useful Docker Compose Commands

Here are some common Docker Compose commands to manage your Domjudge instance:

*   **View container status**:
    ```bash
    sudo docker compose ps
    ```
*   **View logs for all services**:
    ```bash
    sudo docker compose logs
    ```
*   **View logs for a specific service (e.g., domserver)**:
    ```bash
    sudo docker compose logs domserver
    ```
*   **Follow logs in real-time (e.g., judgehost-0)**:
    ```bash
    sudo docker compose logs -f judgehost-0
    ```
*   **Create and start all services defined in `docker-compose.yml`**:
    (If containers already exist, this will recreate them if their configuration or image has changed.)
    ```bash
    sudo docker compose up -d
    ```
*   **Stop running services**:
    (Containers are stopped but not removed.)
    ```bash
    sudo docker compose stop
    ```
*   **Start previously stopped services**:
    ```bash
    sudo docker compose start
    ```
*   **Stop and remove all services, networks, and volumes defined in `docker-compose.yml`**:
    (Use with caution, as this will remove container data unless volumes are explicitly managed outside of compose.)
    ```bash
    sudo docker compose down
    ```
*   **Stop and remove services, networks, and *named* volumes defined in `docker-compose.yml`**:
    ```bash
    sudo docker compose down -v
    ```

## Customization

*   **Number of Judgehosts**: To change the number of judgehosts, you can add or remove `judgehost-X` service definitions in the `docker-compose.yml` file. Ensure each `judgehost` has a unique `DAEMON_ID` and the correct `JUDGEDAEMON_PASSWORD`.
*   **Timezone**: The `domserver` is configured with `CONTAINER_TIMEZONE=Asia/Jakarta`. You can change this to your desired timezone.
*   **Database Credentials**: Database credentials (`MYSQL_ROOT_PASSWORD`, `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_DATABASE`) are defined in the `docker-compose.yml`. While the defaults work for this setup, be mindful of them if integrating with other systems or for production use.
